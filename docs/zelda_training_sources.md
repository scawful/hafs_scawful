# Zelda Training Data Sources

Purpose: keep a curated index of local + external sources and how they should be
used for model training (quality tier, intent, and risk).

## Local Sources (High Signal)
- `~/Code/usdasm`: USDASM (Kan/spannerisms). Gold vanilla source with per-line
  addresses; best for address literacy, debugging, and vanilla reference.
- `~/Code/Assembly/AsarUSALTTPDisassembly`: MathOnNapkins + JaredBrian readable
  disassembly with extra comments; strong for pedagogy + readable routines.
- `~/Code/Oracle-of-Secrets`: Oracle-of-Secrets ROM hack source; primary domain.
- `~/Code/poltergeist`: All Hallows Eve hack; curated, high quality team code.
- `~/Code/Assembly/GuardianBoss` + `~/Code/Assembly/SpiderBoss`: JaredBrian boss
  systems; curated and reusable patterns.
- `~/Code/Assembly/(Euclid)`: small, clean ASM patches (HUD/monologue/palette).
- `~/Code/book-of-mudora`: Oracle docs and notes; good for instruction-style text.
- `~/Code/hyrule-historian`: ROM hack guides; tutorial-style text.
- `~/Code/docs/zelda` + `~/Documents/Zelda`: personal notes and references.
- `~/Code/alttp-hacker-workspace`: mixed hacking resources and disassemblies.
- `~/Code/yaze`, `~/Code/ZScreamDungeon`, `~/Code/hmagic`: editor codebases for
  tool-focused tasks (separate tool-model bucket).

## Local Sources (Legacy, Low Weight)
- `~/Code/Assembly/(Conn)`: legacy hack pack; address-heavy, mixed quality.
- `~/Code/Assembly/(XaserLE)`: overlay/area hacks; readable but older.
- `~/Code/Assembly/(Zarby)`: autogenerated sprite template; use sparingly.
- `~/Code/Assembly/bossheartsv11/ASM Version/Headerless`: well-commented patch.
- `~/Code/Assembly/High Rule Tail`: mixed patch pack; small sample for addresses.

## Local Sources (Research/Do-Not-Train by Default)
- `~/Code/alttp-gigaleak`: valuable for research, but noisy as training data.
- `~/Code/zelda3`: C++ port (logic reference only; not a ROM-hack dataset).
- `~/Code/Assembly` (legacy archive): only include via allowlist and weights.

## Training Intent Mapping
- Vanilla ASM literacy: USDASM + AsarUSALTTPDisassembly.
- Hooking and address pairing: Oracle-of-Secrets + curated hacks.
- Docs/teaching tone: book-of-mudora + hyrule-historian + SNES docs.
- Tooling/code tasks: YAZE + ZScreamDungeon + hmagic (separate model).
- Research-only: gigaleak, zelda3, cut content (use for analysis prompts only).

## Data Quality Notes
- Always keep old hacks behind an allowlist + weights.
- Require $BB:AAAA and $7E:XXXX address mentions in cross-domain samples.
- Track author attribution for style bias (Kan/JaredBrian/Euclid/Zarby89).
