# Model Catalog (hafs_scawful)

Updated: 2025-12-22
This catalog lists model families, specialists, unified models, and name ideas.

## Naming Rules
- Families use the oracle- prefix (oracle-din, oracle-nayru, oracle-farore).
- Specialists do not include the oracle- prefix.
- Unified models are named directly (master-sword, fierce-deity).
- Model IDs can include base model and date suffixes.

## Routing and Families
- oracle-council: MoE router and policy model.
- oracle-din: technical execution (ASM, ROM tooling).
- oracle-nayru: knowledge and documentation.
- oracle-farore: creation and narrative.

## Near-Term Small Models (Priority)
- euclid-asm (1.5B): 65816 ASM specialist.
- Tool-use specialist (3B): command translation for hafs, yaze-mcp, z3ed.
- LSP / code-intel specialist (1.5B or 3B): symbols, xrefs, jumps, errors.

## Specialist Pool (Current and Planned)
- euclid-asm: ASM generation and debugging.
- seph-tilesmith: tiles, sprites, palettes.
- koume-compress: compression (LC_LZ2 and related).
- kotake-decompress: decompression pipelines.
- agahnim-patcher: IPS/BPS and patch workflows.
- kaepora-teacher: code explanation and teaching.
- nayru-lorekeeper: lore and canon.
- farore-questweaver: quest design.
- saria-voice: dialogue and character voice.
- zelda-scribe: UI copy and quest logs.
- gossip-stone: quick lookup and code intelligence.

## Unified Models
- master-sword: unified mid-term model (north star).
- fierce-deity: long-term unified model (successor to master-sword).
